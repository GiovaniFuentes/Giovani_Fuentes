THE IDEA

Project Overview

This project is my attempt to build a Pomodoro-style focus app that helps users manage tasks, breaks, and motivation. I conducted research on 5 popular productivity apps, analyzing features, usability, and user sentiment using Python scripts. The insights from this research shaped the design and functionality of my app.

Key Insights

From analyzing user reviews, I discovered that simplicity is key. Users love clean, easy-to-use interfaces and the ability to customize their experience. Gamification elements are appreciated but must be balanced so they don’t overshadow the main goal: productivity. Finally, stability and minimal bugs are absolute deal breakers—so performance is a top priority.

User Tasks & Goals

The app allows users to:

  • Start and stop focused work sessions (Pomodoro cycles) with adjustable timers.
  • Manage tasks and subtasks to break work into manageable pieces.
  • Receive motivational feedback, like extra break time or visual rewards for completing tasks.

Target Users

My app is designed for students, freelancers, or anyone looking to balance work and breaks effectively. The goal is to require minimal onboarding with an intuitive, straightforward UI, while giving users enough customization to feel in control of their workflow.

Design & Functionality

The main interface is clean and minimal, with a central Start/Break button and optional second window for tasks. Users can customize break times, work-to-break ratios, and even see progress visually through gamification elements like growing trees. Everything is built to run smoothly in the background, with minimal resource usage, and work offline on mobile devices.

Data & Storage

All user data—including tasks, subtasks, due dates, and timer preferences—is stored locally for offline access. JSON or SQLite ensures quick access and reliable storage.

Why This Matters

My goal with this app is not just to provide a tool but to create an experience that encourages focus, rewards progress, and reduces distractions. Every decision—from UI design to performance optimization—is informed by real user feedback and data analysis.

The Process...

The Initial Prototype

We started by sketching and designing a low-fidelity prototype focused on the core idea: a flexible Pomodoro-style timer that visually represents focus and rest. At this stage, the goal was not functionality, but understanding layout, hierarchy, and how users would interpret the interface at a glance.

This initial prototype used a large radial timer as the central interaction point, with minimal text and clear visual emphasis. We intentionally avoided feature overload so we could test whether users understood the app’s purpose without instructions.

Lofidelity Prototypes

We created two low-fidelity prototypes using a “Wizard of Oz” approach, meaning the designs were interactive only in concept. Users were asked to imagine functionality while we observed how they navigated each interface and interpreted its controls/p>

Prototype 1

Prototype 1 focused on simplicity. The main screen featured a large circular (radial) timer with minimal text and a clear work/break toggle. This design emphasized immediate usability, allowing users to quickly understand how to start focusing without feeling overwhelmed.

User testing showed that participants found this version easier to understand and less intimidating. Even users unfamiliar with the Pomodoro technique gravitated toward the large circular timer and felt more confident interacting with it.

Prototype 2

Prototype 2 expanded on functionality by introducing more text, task completion controls, and a rectangular timer design. This version allowed users to complete both main tasks and subtasks, earning additional break time as motivation.

While users appreciated the added information and organizational tools, feedback indicated that this prototype felt more cluttered. Some participants found it harder to quickly understand what actions to take, especially without guidance or visual feedback such as animations or sound.

User Research & Evaluation

We evaluated both prototypes by observing participants as they interacted with each design and then asking follow-up questions. Participants were asked about clarity, ease of navigation, and how motivating or overwhelming the interface felt.

Common feedback highlighted the importance of clear visual cues and immediate understanding. One participant noted, “I didn’t understand what the buttons were—I was just pressing them,” which reinforced the need for clearer labels and better onboarding.

Key Findings

  • Prototype 1 was preferred overall due to its simplicity and clear visual focus.
  • Prototype 2 offered useful features but felt too cluttered for quick interaction.
  • Users wanted more customization options, especially for colors and timer behavior.
  • Participants wanted better feedback through sound, animation, or visual rewards.
  • Not all users were familiar with the Pomodoro technique, indicating the need for a short tutorial.
